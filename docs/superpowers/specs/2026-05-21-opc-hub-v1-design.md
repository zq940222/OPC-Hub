# OPC Hub — v1.0.0 完整版设计规格

**日期：** 2026-05-21  
**状态：** 用户已确认  
**基础版本：** [MVP Foundation 设计](./2026-05-20-opc-hub-design.md)（Plan 1 已完成）

---

## 1. 背景与竞品洞察

### 市场背景

- 全国一人公司存量 **1,600万+**，新注册同比 +47%，每3家新注册企业有1家是 OPC
- 全国已有 105 个 OPC 社区分布 29 座城市，市场需求快速增长
- 银行、政府、互联网平台纷纷切入 OPC 服务赛道

### 竞品差距

| 维度 | 猪八戒网 | 一品威客 | 现有 OPC 社区 | **OPC Hub** |
|------|----------|----------|--------------|-------------|
| 用户定位 | 个人/团队混合 | 个人为主 | 线下空间 | **OPC公司为一等公民** |
| 订单系统 | 竞标模式 | 竞标模式 | 无线上系统 | **发单审核→报名→承接闭环** |
| 财务/法务 | 子公司独立 | 无 | 无 | **聚合入口一站式** |
| 社区 | 弱评价体系 | 无 | 微信群 | **OPC垂直社区+积分** |
| 工具集成 | 无 | 无 | 无 | **AI工具优先工具箱** |

### 竞品借鉴点

- 猪八戒：订单状态透明化展示
- 一品威客：电子合同引导（法务入口钩子）
- 海外平台：积分递减佣金机制（高活跃让利）

## 3. 数据模型补充

在现有 `prisma/schema.prisma` 基础上新增以下模型：

```prisma
model OpcProfile {
  id       String   @id @default(cuid())
  userId   String   @unique
  bio      String?  @db.Text
  skills   String[]
  website  String?
  location String?
  user     User     @relation(fields: [userId], references: [id])
}

model Tool {
  id          String   @id @default(cuid())
  name        String
  description String
  category    String   // "AI工具" | "协作办公" | "财税记账" | "合同法务" | "设计创意"
  url         String
  iconUrl     String?
  embedable   Boolean  @default(false)
  order       Int      @default(0)
  active      Boolean  @default(true)
  createdAt   DateTime @default(now())
}

model PostLike {
  userId    String
  postId    String
  createdAt DateTime @default(now())
  @@id([userId, postId])
}

model Report {
  id         String   @id @default(cuid())
  reporterId String
  targetType String   // "POST" | "COMMENT"
  targetId   String
  reason     String
  resolved   Boolean  @default(false)
  createdAt  DateTime @default(now())
  reporter   User     @relation(fields: [reporterId], references: [id])
}
```

`Order` 模型新增字段：
```prisma
tags  String[]  // OPC能力标签，最多5个
```

`User` 模型新增字段：
```prisma
banned     Boolean      @default(false)
opcProfile OpcProfile?
reports    Report[]
```

## 5. 阶段 1：订单系统

### 5.1 路由清单

| 路由 | 功能 | 权限 |
|------|------|------|
| `/orders` | 订单广场列表 + 筛选 | OPC |
| `/orders/new` | 发布订单表单 | BIZ_OPC |
| `/orders/[id]` | 订单详情 + 报名 | OPC |
| `/orders/[id]/edit` | 编辑订单 | 发布方本人 |
| `/dashboard/orders` | 我的订单（发布/承接） | OPC |

### 5.2 订单广场（`/orders`）

**筛选维度：**
- 行业分类（多选标签）
- OPC 能力标签（多选）：AI开发 / 内容创作 / 财税咨询 / 设计 / 运营 / 法律 / 其他
- 金额范围（预设档：1万以下 / 1–5万 / 5–10万 / 10万以上）
- 截止日期（近7天 / 近30天 / 全部）
- 状态（默认仅显示招募中）

**订单卡片字段：** 标题、发布方（链接主页）、金额、能力标签（最多3个）、已报名人数、截止日期、状态标签

**排序：** 最新发布 / 金额最高 / 报名最少

### 5.3 发布订单（`/orders/new`）

字段：标题、详细描述（富文本）、行业分类、OPC 能力标签（多选，最多5个）、预算金额、截止日期、联系方式（可选）

提交后进入 `PENDING_REVIEW`，页面提示"订单已提交，审核通过后将对所有 OPC 公开"。

### 5.4 订单详情（`/orders/[id]`）

- 完整描述 + 标签 + 金额 + 截止日期
- 发布方信息卡片（链接 OPC 主页）
- **OPC 用户**：填写报名理由（300字以内）→ 提交报名；已报名则显示状态
- **发布方本人**：查看全部报名列表（头像+简介+积分+主页链接），可接受/拒绝单个报名
- 发布方可操作：编辑（仅 DRAFT/PENDING_REVIEW 状态）、关闭、标记完成

### 5.5 我的订单（`/dashboard/orders`）

- Tab「我发布的」：订单列表 + 状态 + 操作按钮
- Tab「我报名的」：订单列表 + 报名状态（待处理/已接受/已拒绝）

### 5.6 状态机与积分触发

```
草稿 ──[提交]──▶ 待审核 ──[管理员通过]──▶ 招募中
                      └──[管理员拒绝]──▶ 已拒绝（通知发布方）

招募中 ──[发布方接受报名]──▶ 进行中
       └──[发布方关闭]──▶ 已关闭

进行中 ──[发布方标记完成]──▶ 已完成（承接方 +100 积分）
```

## 6. 阶段 2：OPC 公司主页

### 6.1 公开展示页（`/profile/[userId]`）

**左栏 — 公司卡片：**
- 头像 + 公司名/姓名
- 加入时间、所在城市
- 能力标签（最多8个）
- 积分 + 平台排名（按积分降序）
- 已完成订单数
- 个人简介（最多300字）
- 个人网站链接
- 关注/取消关注按钮（其他已登录用户）
- 关注数 / 被关注数

**右栏 — 三个 Tab：**
- **承接案例**：已完成订单卡片（标题截断15字，金额显示区间档）
- **发布的帖子**：帖子标题 + 板块 + 点赞数 + 时间
- **关注/粉丝**：头像 + 名称 + 能力标签

本人访问时：右上角显示「编辑主页」按钮。

### 6.2 编辑页（`/settings/profile`）

字段：
- 公司简介（textarea，300字上限）
- 能力标签（多选 + 自定义输入，最多8个）
- 所在城市（省份+城市两级下拉）
- 个人网站（URL）
- 头像上传（腾讯云 COS，最大2MB，正方形裁剪）

使用 Server Action 提交，保存后立即反映到公开主页。

### 6.3 Navbar 头像下拉菜单

```
[头像] 用户名 ▼
─────────────
我的主页      → /profile/[me]
我的订单      → /dashboard/orders
账号设置      → /settings/profile
─────────────
退出登录
```

## 7. 阶段 3：社区系统

### 7.1 板块定义

| 板块 | Slug | 定位 |
|------|------|------|
| 平台公告 | `announcements` | 管理员发布，置顶 |
| AI 工具探讨 | `ai-tools` | 首位展示，最高频需求 |
| 接单经验 | `order-tips` | 报价/沟通/交付技巧 |
| 税务法务 | `tax-legal` | 记账/报税/合规/合同 |
| 经验分享 | `experience` | 创业故事/行业洞察 |
| 问答互助 | `qa` | 快问快答 |
| 闲聊灌水 | `casual` | 轻松话题 |

### 7.2 路由清单

| 路由 | 功能 | 权限 |
|------|------|------|
| `/community` | 首页：板块入口 + 热帖 | OPC |
| `/community/[boardSlug]` | 板块帖子列表 | OPC |
| `/community/post/new` | 发帖 | OPC |
| `/community/post/[id]` | 帖子详情 + 评论 | OPC |
| `/community/messages` | 私信列表 | OPC |
| `/community/messages/[userId]` | 对话页 | OPC |

### 7.3 帖子功能

**发帖表单：** 板块选择 + 标题（最多80字）+ 正文（富文本，支持图片上传到 COS）

**帖子卡片：** 标题、作者（链接主页）、板块、点赞数、评论数、发布时间、置顶/精华标记

**帖子详情：**
- 正文全文渲染
- 点赞（+2积分给作者，防重复）
- 评论列表（时间正序）+ 评论输入框（+3积分给评论者）
- 举报按钮（帖子/评论均可，填写原因写入 `Report`）
- 管理员专属：设为精华（+50积分）、置顶、删除

### 7.4 积分体系

| 行为 | 积分 | 触发时机 |
|------|------|---------|
| 每日首次登录 | +5 | 服务端 session 检测 |
| 发帖 | +10 | 帖子创建成功 |
| 回复评论 | +3 | 评论创建成功 |
| 帖子被点赞 | +2 | 每次被点赞 |
| 帖子被设为精华 | +50 | 管理员操作 |
| 完成订单（承接方） | +100 | 订单状态变为 COMPLETED |
| 置顶帖子（消耗） | −200 | 用户申请置顶 |

所有变动写入 `PointLog`。

### 7.5 私信

**消息列表页：** 按最后消息时间倒序，显示对方头像 + 名称 + 最后消息摘要 + 未读红点

**对话页：**
- 气泡式界面（左对方、右自己）
- 消息输入框 + 发送
- 进入时自动标记已读
- 轮询获取新消息（每5秒，简化实现）

**未读消息数：** Navbar 入口显示红点，服务端渲染。

### 7.6 关注系统

- 在 OPC 主页点击「关注」→ 写入 `Follow` 表
- 主页 Tab 3 展示关注列表和粉丝列表
- 关注动态 Feed 留至 v1.1.0

## 8. 阶段 4：服务页面 + 工具箱

### 8.1 服务页面（`/finance` `/legal` `/banking` `/equipment`）

统一页面结构：
1. Eyebrow + H1 标题 + 服务介绍（2–3段）
2. 「我们能帮你做什么」：4–6条功能点（图标 + 标题 + 一句话描述）
3. 「联系我们」：微信二维码 + 电话号码 + 邮箱（可选）

| 页面 | 功能点方向 |
|------|-----------|
| 财务 (`/finance`) | 代账记账、税务申报、年度审计、发票管理 |
| 法务 (`/legal`) | 合同起草审查、公司注册变更、知识产权、纠纷咨询 |
| 银行服务 (`/banking`) | OPC专属开户、信用贷款、收款结算、理财配置 |
| 设备租赁 (`/equipment`) | 办公设备、摄影器材、生产工具、按需短租 |

联系方式图片存腾讯云 COS，路径通过环境变量配置，管理员后台可替换。

### 8.2 工具箱（`/tools`）

**布局：** 左侧分类标签 + 右侧工具卡片网格（3列）

**分类（顺序固定）：**
1. AI 工具 ⭐（默认选中）
2. 协作办公
3. 财税记账
4. 合同法务
5. 设计创意
6. 全部

**工具卡片：** 图标 + 名称 + 一句话描述 + 分类标签 + 「立即使用」按钮（外链跳转）

**数据来源：** `Tool` 模型，管理员后台配置。

## 9. 阶段 5：管理后台完善

在现有（订单审核 + 子账号管理）基础上新增：

### 9.1 用户管理

- 表格：用户名、邮箱/手机、角色、积分、注册时间、封禁状态
- 操作：修改角色（OPC ↔ BIZ_OPC）、封禁/解封（`User.banned`，middleware 拦截封禁账号）
- 搜索：按名称/手机/邮箱

### 9.2 内容管理（举报处理）

- 举报列表（`Report` 表）：类型 + 内容摘要 + 举报原因 + 举报人
- 操作：删除内容、标记举报已处理
- 筛选：未处理 / 已处理

### 9.3 工具配置

- 工具列表：名称 + 分类 + 启用状态
- 操作：新增、编辑、启用/禁用、调整顺序（上移/下移）
- 表单字段：名称、描述、分类、URL、图标URL、是否可嵌入

### 9.4 子账号权限结构（明确定义）

```typescript
type AdminPermissions = {
  users: boolean         // 用户管理
  orders: boolean        // 订单审核
  content: boolean       // 内容管理（举报处理）
  tools: boolean         // 工具配置
  announcements: boolean // 公告发布
}
```
